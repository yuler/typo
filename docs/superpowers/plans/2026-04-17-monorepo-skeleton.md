# Monorepo Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the existing Tauri desktop app into `apps/desktop/` and set up minimal `pnpm` workspace tooling at the repo root, with no behavior changes to the app itself.

**Architecture:** Single-branch, multi-commit migration. Files move via `git mv` (preserves rename history), then configs are rewritten to wire the new workspace. Every step stages cleanly against the previous one; verification runs at the end before pushing. Optional squash collapses intermediate commits into a single reviewable commit.

**Tech Stack:** pnpm 10.33 workspaces, Node (version pinned in `.nvmrc`), Tauri 2, Vue 3 / Vite, Rust (unchanged), GitHub Actions, `tauri-apps/tauri-action@v0`.

**Spec:** `docs/superpowers/specs/2026-04-17-monorepo-skeleton-design.md`

**Working branch:** `refactor/monorepo` (already checked out)

**Note on TDD:** This migration has no application logic to test. The functional test is the verification suite in Task 10: the app must install, lint, build, dev, and bundle exactly as before. Each content-change task includes a shape-level check (jq / rg / syntax validation) to catch obvious errors early.
